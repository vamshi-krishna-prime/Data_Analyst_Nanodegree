0.1 Introduction:¶

Over the past decade, bicycle-sharing systems have been growing in number and popularity in cities across the world. Bicycle-sharing systems allow users to rent bicycles on a very short-term basis for a price. This allows people to borrow a bike from point A and return it at point B, though they can also return it to the same location if theyd like to just go for a ride. Regardless, each bike can serve several users per day. Thanks to the rise in information technologies, it is easy for a user of the system to access a dock within the system to unlock or return bicycles. These technologies also provide a wealth of data that can be used to explore, how these bike-sharing systems are used.

The project make use of Python to explore data related to bike share systems, provided by Metro Bike Share. This data set includes information about individual rides made in a bike-sharing system covering the greater Los Angeles area.

0.5 Dataset:¶

Dataset Link: bikeshare.metro.net

This data set includes information about individual rides made in a bike-sharing system covering the Los Angeles area.

Each trip is anonymized and includes:

  • trip_id: Locally unique integer that identifies the trip
  • duration: Length of trip in minutes
  • start_time: The date/time when the trip began, presented in ISO 8601 format in local time
  • end_time: The date/time when the trip ended, presented in ISO 8601 format in local time
  • start_station: The station ID where the trip originated (for station name and more information on each station see the Station Table)
  • start_lat: The latitude of the station where the trip originated
  • start_lon: The longitude of the station where the trip originated
  • end_station: The station ID where the trip terminated (for station name and more information on each station see the Station Table)
  • end_lat: The latitude of the station where the trip terminated
  • end_lon: The longitude of the station where the trip terminated
  • bike_id: Locally unique integer that identifies the bike
  • plan_duration: The number of days that the plan the passholder is using entitles them to ride; 0 is used for a single ride plan (Walk-up)
  • trip_route_category: "Round Trip" for trips starting and ending at the same station or "One Way" for all other trips
  • passholder_type: The name of the passholder's plan
  • bike_type: The kind of bike used on the trip, including standard pedal-powered bikes, electric assist bikes, or smart bikes.

def extract_filename(url, company):
    year_regex = re.compile(r"(\d{4})-([q]\d)|([Q]\d)_(\d{4})")
    matching_object = year_regex.search(url)
    if matching_object.group(1) == None:
        year = "{}_{}".format(matching_object.group(4), matching_object.group(3)).replace('Q', 'q')
    else:
        year = "{}_{}".format(matching_object.group(1), matching_object.group(2))
    return company + '_' + year


def get_response(url, filename, attempt=1, max_attempts=5):
    '''
    The default User-Agent of requests is python-requests, and in some cases websites don't like traffic 
    from "non-browsers", so they try to block such traffic. This can be avoided by making the request appear 
    like coming from a browser, by adding an extra header parameter.
    '''
    headers = {'User-Agent': 'Mozilla/5.0 (Windows NT 6.1; WOW64) AppleWebKit/537.36 (KHTML, like Gecko)\
                Chrome/56.0.2924.76 Safari/537.36'}
    # add time gap between every request to avoid timeout error
    time.sleep(5)
    response = requests.get(url, headers=headers, stream=True, allow_redirects=False, timeout=10)
    try:
        requests.post(url, headers, timeout=10)
    except requests.exceptions.Timeout:
        print("Timeout occurred")
        if attempt <= max_attempts:
            return get_response(url, filename, attempt=attempt+1)
    # check the status of the request
    try:
        response.raise_for_status()
        print('{}'.format(filename).ljust(20, ' '), ':', 'Data request successful.')
    except Exception as exc:
        print('{}'.format(filename).ljust(20, ' '), ':', 'WARNING - "{}"'.format(exc))
        if attempt <= max_attempts:
            return get_response(url, filename, attempt=attempt+1)
    return response


def extract_csv(response):
    try:
        zf = ZipFile(BytesIO(response.content))
        # find the first matching csv file in the zip:
        match = [s for s in zf.namelist() if ".csv" in s][0]
        print('CSV file obtained'.ljust(20,' '), ':', match)
        df = pd.read_csv(zf.open(match), low_memory=False)
        return df
    except BadZipFile:
        print('BadZipFile')


def save_csv_file(df, filename):
    csv_filename = '{}.csv'.format(filename)
    # search for any files which match the filename
    files_present = glob.glob(csv_filename)
    # if no matching files exist, write to csv. else, print warning
    if not files_present:
        df.to_csv(csv_filename, sep=',', index= False)
        print('Action performed'.ljust(20, ' '), ': CSV file saved')
    else:
        print('Warning'.ljust(20, ' '), ': This file already exists!!')

def estimate_fare(df):
    df.loc[df['passholder_type'] == 'Walk-up','fare'] = df.query(' passholder_type == "Walk-up" ')['duration'].apply(lambda x: np.add(np.multiply(np.maximum(0, (math.ceil(x/30)-1)), 1.75),1))
